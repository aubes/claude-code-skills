Scope: **comment conciseness and content** (what a comment says and how long). Tone/humor is out of scope.

When writing code comments:

- Be concise: one short sentence max, no filler words
- Don't explain how the framework or language works; mention its behavior only when it justifies an architectural decision or a non-obvious choice
- If the code is self-explanatory, don't add a comment at all
- Structured docblock annotations (`@param`, `@return`, `@throws`...) are exempt from the one-sentence limit; their prose summary is not
