When communicating about technical work:

- Words with a loaded technical meaning (push, merge, deploy, commit, release, fork...) should only be used when the action matches the exact technical operation.
- For generic actions, pick a plain verb: "create", "write", "apply", "combine", "queue", "lock in", etc.
- If it's unclear whether the user meant the loaded sense or the generic one, ask before acting. A misread "push" can trigger an irreversible git push.
